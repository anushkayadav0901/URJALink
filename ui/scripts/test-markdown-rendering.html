🔍 Markdown Rendering Issue - Diagnosis

🐛 Problem Identified

Root Cause: The API is returning markdown text WITHOUT any newline characters (\n). All content is concatenated into a single line, which breaks markdown parsing.

Impact: Headers, lists, and paragraphs are not rendered correctly because markdown requires line breaks to separate elements.

🔧 Recommended Fix

The backend API needs to ensure newline characters are included in the streamed response. There are two potential solutions:

Solution 1: Fix at Backend (Recommended)

The backend should preserve newlines when streaming the LLM response. Check if:

  • The LLM is generating text without newlines
  • The SSE streaming code is stripping newlines
  • The response encoding is removing line breaks

Solution 2: Fix at Frontend (Workaround)

Add a preprocessing step in the frontend to insert newlines before markdown markers:

const fixMarkdownLineBreaks = (text) => {
  return text
    // Add newlines before headers
    .replace(/(#+\s)/g, '\n$1')
    // Add newlines before list items
    .replace(/([.!?])([-*]\s)/g, '$1\n$2')
    .replace(/([.!?])(\d+\.\s)/g, '$1\n$2')
    // Add spacing around bold/italic
    .replace(/([.!?])(\*\*)/g, '$1\n$2')
    .trim();
};
